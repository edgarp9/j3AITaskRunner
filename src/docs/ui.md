# UI 문서

Tkinter 화면 구조, 다이얼로그/팝업, DPI, UI 문구, 메인 스레드 갱신 기준을 확인할 때 읽는다. 플랫폼별 DPI와 프로세스 제약은 [`platform-notes.md`](platform-notes.md)를 함께 확인한다.

## 화면 구조

- 워크스페이스 헤더에는 큐 상태, 큐 시작/중지 토글, `새 세션`, `새 프리셋`, `가져오기` 제어를 둔다.
- 공유큐 상태에서는 모든 워크스페이스 헤더가 같은 공유큐 상태와 토글 동작을 표시한다.
- 워크스페이스 화면은 세션 본문, 오른쪽 작업 목록, 왼쪽 사이드바 하단 설정 버튼과 설정 요약을 제공한다.
- 세션 본문은 `프롬프트`, `진행 로그`, `이력` 탭을 제공한다. 부모 프리셋 세션은 `후보` 탭을 추가한다.
- `진행 로그`는 provider 로그 원문을 유지하되 JSONL 문자열 안의 이스케이프된 줄바꿈은 실제 줄바꿈으로 표시한다.
- `이력`은 실행 프롬프트와 완료 응답을 턴 단위로 표시하고 실패 원인은 같은 턴의 `Error` 섹션에 표시한다.
- 사이드바/메인, 세션 탭/우측 작업 목록 경계는 드래그로 조절할 수 있어야 한다. 접힌 사이드바는 폭 0이며 빈 버튼 바를 남기지 않는다.
- 워크스페이스 작업 목록은 `순서`, `세션`, `상태`, `프롬프트` 컬럼만 표시한다. 작업 ID는 내부 선택/삭제에만 사용한다.
- 실행 중 작업이 있는 워크스페이스 탭과 세션 탭은 녹색으로 식별 가능해야 하며, 모든 작업이 종료되면 제거한다.
- 기본 화면, 설정 요약, 설정 대화상자, 목록, 작업 테이블, 텍스트 영역은 같은 다크 테마 색상 체계를 사용한다.

## 세션 상단 표시

- 오른쪽 AI 정보 묶음은 `AI설정`, 실행 옵션 요약, `훅`, `바로실행`, `세션 닫기` 순서다.
- 실행 옵션 요약은 `AI 실행기 / model / 추론레벨` 형식이다. provider 기본값은 현재 언어의 `자동` 라벨로 표시한다.
- `AI설정` 팝업에서 저장은 선택값을 반영하고, 취소/창 닫기/`Esc`는 변경 전 값을 유지한다.
- `훅` 팝업에서 저장은 사용 여부, 실행파일 경로, 줄 단위 arguments를 세션 런타임 상태에 반영하고, 취소/창 닫기/`Esc`는 변경 전 값을 유지한다.
- `바로실행`은 일반 세션에서만 활성화한다. 프리셋/후보 세션, 같은 세션에 대기/설정 필요/실행 중 작업이 있는 경우, 바로실행 등록 처리 중인 경우에는 비활성화한다.
- `실행 중` 문구에는 현재 실행 중 job과 같은 세션의 완료 job 번호를 괄호에 표시한다.
- `종료` 문구에는 완료 job 번호와 실패/취소/타임아웃/설정 사용자 메시지를 같은 줄에 짧게 표시한다.
- 새 대기 작업이 선택된 상태라면 `종료` 대신 `대기중: job-N`을 표시한다.
- 보조 메시지가 없을 때는 빈 줄을 예약하지 않는다.
- 작업 등록 후에는 `진행 로그` 탭으로, manual 후보 선택 이벤트 후에는 부모 프리셋의 `후보` 탭으로 자동 전환한다.

## 프리셋 UI

- 프리셋 세션은 `Language`, `Instruction`, `Work Priority`, prefix 편집 영역, 자동커밋 체크박스를 제공하고 일반 프롬프트 에디터는 숨긴다.
- `Work Priority=manual`은 `워크스페이스 개별큐`일 때만 표시한다. `공유큐`에서는 숨기고 이미 선택된 값은 `medium` 같은 기본값으로 되돌린다.
- 등록줄은 `Auto Commit -> AI설정 -> 실행 옵션 요약 -> 등록` 순서다.
- 프리셋 입력 콤보박스와 AI 설정/요약은 남는 가로 폭을 채우지 않고 압축된 고정 폭으로 왼쪽 정렬한다.
- 분석 작업 등록 대기 또는 등록 뒤에는 관련 입력, 자동커밋, AI 설정, 등록 버튼을 비활성화한다. 실행 옵션 요약은 잠긴 값 확인을 위해 계속 표시한다.
- `후보` 탭은 빈 상태, manual 후보 선택 대기, Continue 후 처리 중 상태를 구분한다.
- manual 후보 목록은 candidate id, title, priority, problem, risk, impact, evidence를 체크박스 목록으로 표시한다.
- 하나 이상 선택되기 전에는 `Continue`를 비활성화하고, 누르면 버튼과 체크박스를 즉시 비활성화한다.

## 다이얼로그와 편집 메뉴

- 팝업과 설정 대화상자는 모달로 띄우고 메인 창 기준 중앙에 배치한다. 음수 좌표 모니터에서도 좌표를 0으로 보정하지 않는다.
- 가져오기 모달은 여러 줄 `Text`, 자동커밋 체크박스, `Step 실행 방식` 드롭다운, `등록` 버튼을 제공한다.
- 프롬프트 원문 보기 창은 중앙 배치하고 스크롤 가능한 텍스트 영역에 원문을 표시한다.
- 프롬프트 입력, 프리셋 prefix, 가져오기 입력은 우클릭 메뉴로 잘라내기/복사/붙여넣기/모두 선택을 제공한다.
- 진행 로그와 이력 출력 영역은 우클릭 메뉴로 복사/모두 선택을 제공한다.
- 설정 대화상자의 `작업큐 방식`은 `워크스페이스 개별큐`와 `공유큐`다. 실행 중 작업이 있으면 비활성화하고 저장도 막는다.
- About 창에는 앱 버전, 소스 코드 링크, `about.txt` 본문을 표시한다.
- `Licenses` 창에는 `Notice Inventory`만 표시하고 내부 `Release Checklist`는 표시하지 않는다.

## Tkinter 구현 제약

1. Tkinter 위젯 조작은 메인 스레드에서만 수행한다.
2. 외부 프로세스 실행, 로그 수집, 파일 저장은 UI 이벤트 핸들러에서 블로킹하지 않는다.
3. 백그라운드 결과는 `queue.Queue`와 `after(...)` 루프로 메인 스레드에 전달한다.
4. manual `Continue`는 UI 이벤트 핸들러에서 후속 작업을 직접 만들지 않고 런타임 action으로 전달한다.
5. 탭이 많아져도 UI 전체를 매번 다시 그리지 말고 필요한 영역만 부분 갱신한다.
6. 실시간 로그는 사용자가 하단을 보고 있거나 로그가 비어 있을 때만 하단 추적을 유지한다.
7. 창 종료 시 실행 중 작업, 폴링 루프, 예약된 `after` 콜백을 정리한다.

## UI 문구

- 화면 문구는 짧은 작업형 표현을 우선한다. 예: `등록`, `진행 로그`.
- 내부 상태명 `설정입력대기`는 UI에서 `설정 필요`로 표시한다.
- 로그 파일 저장 옵션은 `파일 로그 저장`으로 표시해 실시간 로그와 구분한다.
- 프리셋 입력 라벨은 화면에서 `언어`, `지시문`, `우선순위`로 표시하고 내부 값은 `Language`, `Instruction`, `Work Priority`를 유지한다.
- 오류 메시지는 내부 원인 전체가 아니라 사용자가 확인할 대상만 짧게 안내한다.
