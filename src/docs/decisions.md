# 결정사항 문서

현재도 유효한 기술 결정과 운영 규칙의 배경을 확인할 때 읽는다. 과거 변경 이력이나 완료 기록은 남기지 않는다.

## 기술 스택

- 데스크톱 UI는 Python 표준 Tkinter를 기본으로 유지한다.
- 소스 실행과 기본 unittest는 표준 라이브러리와 Tkinter만으로 동작해야 한다.
- CI 기준 Python은 3.12다. 로컬 버전 차이는 [`build-and-test.md`](build-and-test.md)의 기준으로 해석한다.

## 계층과 구조

- 기본 의존 방향은 `ui -> app -> infra/domain`이다.
- `domain`은 I/O 없이 검증 가능한 모델과 정책을 소유한다.
- `ui`는 Tkinter 위젯 조작과 사용자 이벤트 전달에 집중하고, 파일 I/O와 외부 프로세스 실행을 직접 수행하지 않는다.
- 큰 facade는 공개 import 경로를 유지하되 책임별 helper 모듈로 나눈다. 현재 facade 기준은 [`architecture.md`](architecture.md)를 따른다.
- 불필요한 인터페이스, 팩토리, 전략 패턴, 제네릭 레이어, 래퍼, 설정 기반 구조를 새로 도입하지 않는다.

## 실행 모델

- 외부 AI 실행기는 provider CLI를 별도 프로세스로 실행한다.
- 자동 실행 경로는 provider의 비대화형/스크립트용 명령을 사용하고 대화형 TUI를 큐 실행 경로로 사용하지 않는다.
- provider stdout은 파일 로그 저장 옵션과 무관하게 UI 진행 로그에 표시한다.
- timeout, 사용자 취소, 프로세스 트리 종료, 마지막 응답 판정은 provider 공통 계약으로 유지한다.

## 큐와 상태

- 큐 방식은 `per_workspace`와 `shared`를 유지한다.
- `per_workspace`는 워크스페이스마다 실행 슬롯 1개를 갖고, `shared`는 열린 모든 워크스페이스가 전역 실행 슬롯 1개를 공유한다.
- 일반 세션 `바로실행`은 일반 큐/공유큐 슬롯을 점유하지 않는다.
- 작업 상태와 큐 규칙의 상세 계약은 [`features.md`](features.md)가 소유한다.

## 저장과 복원

- 저장되는 것은 앱 설정과 저장된 워크스페이스 목록이다.
- 작업, 큐, 세션 탭, 세션 턴 이력, 세션 종료 훅 설정은 앱 재시작 후 복원하지 않는다.
- 저장된 워크스페이스 제거는 저장 목록에서만 제거하며 실제 폴더, 열린 탭, 세션, 작업은 삭제하지 않는다.

## 릴리즈와 고지

- 릴리즈 빌드는 PyInstaller `onedir` 방식만 사용한다.
- Windows와 Linux 빌드 venv는 공유하지 않는다.
- 공개 바이너리 배포 시 해당 릴리스에 대응하는 전체 소스 제공 방식을 함께 준비한다.
- 라이선스와 고지 기준은 [`license.md`](license.md), 빌드 산출물 기준은 [`release.md`](release.md)를 따른다.
